Feature/Change: Kudos Board — a recognition system with chat-message giving + **MongoDB Atlas** persistence. **Give:** chat `kudos @member [reason]` (NOT a slash command) — up to 7 per giver per day (resets midnight GMT+7), multi-mention supported, no self/bot kudos, optional reason stored. Feature/Change: Party Finder — Join is CLASS-BASED again, now multi-role (replaces the brief "any role freely switchable" experiment; switching is kept but constrained to a class's allowed roles). Each class maps to an ordered list of categories it may apply for: Knight → Tank/DPS, Priest → DPS/Heal, all other classes (Assassin/Blacksmith/Druid/Gunslinger/Hunter/Wizard) → DPS only. A member must hold a class role to join, may join/switch only to categories their class can fill (e.g. Knight switches Tank↔DPS, a DPS class clicking Tank is refused with "you can apply as: DPS"), and the existing switch/full/already-in-slot behavior is unchanged otherwise. Leader: must hold a class role to start; their party-card slot is auto-filled into the first of their class's roles (in listed order) that has an open slot; the Roles form must include at least one slot a leader's class can fill. Druid is confirmed DPS. No file deletions.
